# Guinea Pig

![Guinea Pig](guinea-pig.png)

## Description
This repo is a **security-scanner test bed**: intentionally-vulnerable fixtures,
**one CWE per directory**, each paired with its remediation, plus a running web
service for dynamic (DAST) testing.

### Layout
```
static/CWE-<n>/<language>/vulnerable.<ext>   # vulnerable fixture — the path IS the ground truth
static/CWE-<n>/<language>/patched.<ext>      # the remediation (a negative: expect no finding)
static/CWE-798/catalogue/<provider>/*.txt    # per-rule secret examples (positives — secret-scanner recall)
static/CWE-798/false-positives/*             # benign code that resembles a rule (negatives — precision)
pipeline/sca/<eco>/<manifest>                # vulnerable dependency manifests (SCA)
pipeline/ci/<platform>/...                   # realistic CI files (plumber / poutine)
dynamic/                                     # runnable Express service for DAST (+ openapi.yml)
```
`static/CWE-89/python/vulnerable.py` is expected to be flagged as CWE-89.

- **static / SAST** — languages: java, c, cpp, csharp, python, javascript
- **static / IAC** — languages: terraform, ansible, kubernetes
- **static / PIPELINE** — languages: github, gitlab (per-CWE snippets)
- **static / SECRET** — env, ini, conf, pem (hard-coded secrets / private key), plus
  a **catalogue** of 1028 per-rule secret examples and 21 **false-positive** look-alikes
  under `static/CWE-798/` (see their `README.md`s)
- **pipeline / SCA + CI** — `pipeline/sca/` (vulnerable deps) and `pipeline/ci/`
  (whole `.gitlab-ci.yml` / `.github/workflows/`) — see `pipeline/README.md`
- **dynamic / DAST** — `dynamic/` Express app, one CWE-annotated route per endpoint

Each fixture's detecting scanner family (SAST/IAC/PIPELINE/SECRET/DAST/SCA) is
recorded in the catalog (`scanner_types`).

### Reports & benchmark
- `reports/vulnerabilities.json` — the **structured ground truth**: positives
  (`findings` + the secret `secret_catalogue`) and negatives (`false_positives`).
  Generated by `reports/build_report.py` (re-run after adding fixtures).
- `reports/INVENTORY.md` — human recap: CWE × language matrix and totals.
- `reports/VULNERABILITIES.md` — schema + update procedure.
- `benchmark/` — runs scanners and compares their findings against that ground
  truth, reporting **detection coverage** (recall over positives) and **errors**
  (false positives on `patched.*` / look-alikes, plus noise):
  ```bash
  python3 reports/build_report.py                                   # refresh ground truth
  python3 benchmark/run.py --dry-run                                # inspect it
  python3 benchmark/run.py --mcp ../security_mcp --tools opengrep,trivy-misconfig,gitleaks
  ```
- `dynamic/README.md` — how to run and scan the DAST target, plus exposure warnings.


## VSCode configuration
The MCP server is declared in `.vscode/mcp.json` (`security-mcp-server` →
`http://127.0.0.1:8000/mcp/`). Open VSCode's MCP **Servers** view and **Start** it.
You may need to be signed in to GitHub Copilot for it to work.

## Claude code configuration 
```
claude mcp add --transport http --scope project my-serveur http://127.0.0.1:8000/mcp/
```

## Sources
- **SAST / SCA base fixtures** — adapted from
  https://github.com/dehvCurtis/vulnerable-code-examples
- **Hard-coded-secret config fixtures** (`env`, `ini`, `conf`, `pem`) — drawn from
  https://github.com/Yelp/detect-secrets/tree/master/test_data
- **CWE-798 secret catalogue** (`static/CWE-798/catalogue/`, 1028 per-rule examples)
  — generated from the regex rule packs of kingfisher, titus, noseyparker,
  gitleaks and betterleaks in the sibling `../secret-rules` repo. The benign
  look-alikes in `static/CWE-798/false-positives/` are hand-written.
